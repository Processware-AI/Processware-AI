## 2. Problem — 자동차 SW 팀이 매일 부딪히는 현실

### ❶ ASPICE 산출물 35종, 개발자 8명이 코딩을 멈춥니다

요구사항 명세서, 아키텍처 설계서, 상세설계서, 단위 테스트 케이스,
통합 테스트 케이스, 추적성 매트릭스 —
심사에 필요한 산출물은 **35종 이상**.
개발자들이 코드 에디터 대신 Word를 열고 있는 시간이 **주당 20시간**을 넘깁니다.

### ❷ ASPICE + ISO 26262 + ISO 21434, 같은 내용을 3번 씁니다

ASPICE SWE.1의 소프트웨어 요구사항과 ISO 26262 Part 6의 안전 요구사항은
**60% 이상 중복**됩니다. 여기에 ISO 21434의 사이버보안 요구사항까지 더하면,
통합 설계 없이는 동일한 내용을 표준별로 따로 작성하는 삼중 작업이 발생합니다.
이 비효율을 인식하면서도 "어디가 겹치는지" 정리할 여력이 없는 팀이 대부분입니다.

### ❸ SEPG 담당자 1명이 퇴사하면, 다음 심사가 흔들립니다

프로세스 지식이 특정 개인에게 집중되어 있어,
핵심 인력 퇴사 시 **평균 6~9개월의 역량 공백**이 발생합니다.
신규 담당자는 ASPICE 4.0 변경사항부터 다시 학습해야 하고,
외부 컨설팅을 재의뢰하면 다시 수천만 원의 비용이 필요합니다.

## 3. Services — 자동차 SW 팀을 위한 AI Agent 3종

### 🤖 표준 통합 갭 분석 Agent

ASPICE · ISO 26262 · ISO 21434, 3개 표준의 요구사항을
귀사의 현행 프로세스와 대조하여 **통합 갭 분석 리포트**를 생성합니다.
표준 간 중복 항목은 자동으로 매핑하여, 이중 작업 영역을 식별합니다.

> 갭 분석 소요 **4주 → 3일**, 표준 간 중복 항목 **자동 식별**

### 📄 산출물 자동 생성 Agent

요구사항을 입력하면, ASPICE Work Product 양식에 맞는
설계서·테스트 케이스·추적성 매트릭스를 자동 생성합니다.
ISO 26262의 안전 요구사항과 ISO 21434의 보안 요구사항이
동일 산출물 내에서 통합 추적됩니다.

> 산출물 작성 시간 **70% 단축**, 추적성 불일치 오류 **90% 감소**

### 🎓 모의 심사 & 팀 교육 Agent

AI가 ASPICE 심사원 역할로 모의 심사를 진행합니다.
"이 Work Product의 근거를 설명해주세요"부터
"SWE.2와 SWE.3 간 일관성을 어떻게 확보했습니까?"까지 —
실제 심사에서 나오는 질문 패턴으로 팀의 약점을 사전에 잡습니다.

> 교육 참여율 대면 교육 대비 **2.5배**, 첫 심사 통과율 **95%**
