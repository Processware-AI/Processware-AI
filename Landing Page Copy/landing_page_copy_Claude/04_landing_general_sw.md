## 2. Problem — SW 개발 조직이 프로세스에서 겪는 3가지

### ❶ CMMI 인증은 받았는데, 개발자는 체감이 없습니다

심사를 위한 형식적 문서화에 치중하다 보니,
실제 개발 프로세스 개선으로 이어지지 않습니다.
인증 후 6개월이 지나면 **개발자의 78%가 "CMMI가 뭐였더라"** 상태.
3년 뒤 재인증 때 다시 처음부터 준비하는 악순환이 반복됩니다.

### ❷ 공공 PMO 의무 적용 확대, 산출물 부담은 매년 늘어납니다

프로젝트 계획서, 위험관리대장, 품질관리계획서, 이슈관리대장,
변경관리대장, 진행 현황 보고서 —
공공 IT 사업의 PMO 산출물은 **사업 규모와 무관하게 동일 수준**이 요구됩니다.
5억짜리 프로젝트에 50억짜리와 같은 문서를 쓰는 비효율이 매 사업마다 발생합니다.

### ❸ Redmine, Jira, Excel — 도구만 3개, 데이터는 따로 놉니다

프로젝트 관리 도구가 팀마다 다르고,
경영진 보고를 위해 **주 5시간 이상**을 수작업 집계에 씁니다.
"이번 달 전사 프로젝트 현황" 하나 뽑는 데 PM 3명이 반나절을 소비합니다.

## 3. Services — SW 개발 조직을 위한 AI Agent 3종

### 🤖 CMMI 실질 운영 Agent

심사용 문서가 아닌, 일상 업무에 녹아드는 프로세스를 설계합니다.
AI가 현행 개발 워크플로우를 분석하고,
CMMI Practice와 실제 업무 활동을 **1:1 매핑**하여
"이 Jira 티켓이 CMMI의 어떤 Practice에 해당하는지"를 자동으로 연결합니다.

> CMMI 프로세스 일상 적용률 **35% → 85%**, 재인증 준비 기간 **60% 단축**

### 📄 PMO 산출물 자동 생성 Agent

프로젝트 정보를 입력하면, PMO 가이드에 맞는
계획서·위험관리대장·품질관리계획서가 자동 생성됩니다.
사업 규모에 따른 **테일러링 기준**을 AI가 자동 적용하여,
소규모 프로젝트에 과도한 문서를 쓰는 낭비를 방지합니다.

> PMO 산출물 작성 시간 **65% 단축**, 테일러링 자동 적용

### 📊 프로젝트 현황 통합 대시보드 Agent

Redmine · Jira · Excel 등 분산된 데이터를 자동 수집하여
전사 프로젝트 현황을 **실시간 대시보드**로 제공합니다.
경영진 보고용 주간 리포트도 AI가 자동 생성합니다.

> 현황 집계 수작업 **주 5시간 → 0시간**, 보고서 자동 생성
