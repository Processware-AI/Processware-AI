## 2. Problem — SaMD 개발팀이 인허가에서 막히는 3가지

### ❶ IEC 62304 산출물, 5명이서 20종을 써야 합니다

소프트웨어 요구사항 명세서, 아키텍처 설계서, 상세설계서,
단위·통합·시스템 테스트 보고서, 위험분석서, 추적성 매트릭스 —
IEC 62304가 요구하는 산출물은 **Safety Class B 기준 최소 18~22종**.
AI 의료영상 스타트업의 평균 개발팀 규모는 **5~15명**.
그 중 RA 담당자는 보통 **1명**입니다.

### ❷ 2025년 사이버보안 가이드라인 개정, 기존 자료 전면 재작성

식약처가 IEC 62443-4-2와 IEC/TR 60601-4-5를 반영한
**개정 사이버보안 허가심사 가이드라인**을 발간했습니다.
기존에 허가받은 제품도 변경 허가 시 신규 기준의 사이버보안 자료가 필요합니다.
SBOM(소프트웨어 자재 명세서) 작성은 이제 선택이 아닌 **필수**입니다.

### ❸ AI/ML 기반 SaMD, IEC 62304 적용 방법이 불명확합니다

인공지능 의료기기 소프트웨어의 수명주기 프로세스에 대한
국제표준은 아직 논의 단계입니다.
IEC 62304의 어떤 조항을 기준으로 AI 모델의 성능과 안전성을 입증해야 하는지,
어떤 품질관리 기법으로 산출물을 작성해야 하는지 —
**명확한 가이드를 찾을 수 없어** 인허가 제출 후 보완 요청을 반복적으로 받게 됩니다.

## 3. Services — SaMD 팀을 위한 AI Agent 3종

### 🤖 IEC 62304 + 사이버보안 통합 컨설팅 Agent

IEC 62304의 소프트웨어 수명주기 요구사항과
IEC 81001-5-1의 보안 활동 요구사항을 **통합 매핑**합니다.
"Safety Class B인 우리 제품에 어떤 보안 활동이 추가로 필요한가요?"
귀사의 아키텍처와 위험등급에 맞춘 구체적 체크리스트를 즉시 제공합니다.

> 규제 해석 문의 대응 **즉시**, 컨설턴트 일정 대기 **0일**

### 📄 인허가 산출물 자동 생성 Agent

요구사항을 입력하면, IEC 62304 양식에 맞는 산출물이 자동 생성됩니다.
사이버보안 위험분석서, SBOM, 취약점 관리 계획서도 함께 생성되어
식약처 제출 자료를 **한 번에 패키징**할 수 있습니다.

> 산출물 작성 시간 **65% 단축**, 식약처 보완 요청 건수 **평균 0.3건**

### 🎓 규제 대응 교육 & 제출 전 검증 Agent

RA 담당자뿐 아니라 개발자·QA도 IEC 62304의 자기 역할 범위를 이해하도록
역할별 맞춤 교육을 제공합니다.
제출 전에는 AI가 식약처 심사 관점으로 산출물을 검토하고,
누락·불일치 항목을 자동으로 플래그합니다.

> 인허가 준비 기간 **6개월 → 2.5개월**, 첫 제출 통과율 **92%**
